# DocWen

Программное обеспечение для конвертации форматов документов и диаграмм - поддерживает двустороннюю конвертацию Word/Markdown/Excel. Работает локально, обеспечивая безопасность и надежность данных.

## 📖 История проекта

Это программное обеспечение изначально было разработано для повседневной работы печатного отдела для решения следующих проблем:
- Форматы документов, присылаемых различными отделами, хаотичны и требуют приведения к стандартным форматам.
- Существует множество типов документов, каждый из которых имеет свои фиксированные требования к формату.
- Требуется работа оффлайн, адаптация к внутренней сети и старому оборудованию.

**Философия дизайна**: Это программное обеспечение позиционируется как легкий и простой инструмент. Хотя оно не может сравниться с профессиональными инструментами, такими как LaTeX или Pandoc, по профессионализму и функциональной полноте, оно выигрывает за счет нулевой стоимости обучения и готовности к использованию, что делает его подходящим для повседневных офисных сценариев, где требования к формату не слишком строгие.

## ✨ Основные функции

- **📄 Конвертация формата документа** - Двусторонняя конвертация Word ↔ Markdown. Поддерживает конвертацию математических формул и двустороннюю конвертацию разделителей (три типа разделительных линий Markdown против разрывов страниц, разрывов разделов и горизонтальных линий Word). Поддерживает форматы DOCX/DOC/WPS/RTF/ODT и другие.
- **📊 Конвертация формата таблиц** - Двусторонняя конвертация Excel ↔ Markdown. Поддерживает форматы XLSX/XLS/ET/ODS/CSV. Включает инструменты сводки таблиц.
- **📑 PDF и файлы макетов** - Конвертация PDF/XPS/OFD в Markdown или DOCX. Поддерживает слияние, разделение PDF и другие операции.
- **🖼️ Обработка изображений** - Поддерживает двустороннюю конвертацию и сжатие форматов JPEG/PNG/GIF/BMP/TIFF/WebP/HEIC.
- **🔍 Распознавание текста OCR** - Интегрирован RapidOCR для извлечения текста из изображений и PDF.
- **✏️ Корректура текста** - Проверяет опечатки, пунктуацию, символы и чувствительные слова на основе пользовательских словарей. Правила можно редактировать в интерфейсе настроек.
- **📝 Система шаблонов** - Гибкий механизм шаблонов, поддерживающий пользовательские форматы документов и отчетов.
- **💻 Двухрежимная работа** - Графический интерфейс пользователя (GUI) + Интерфейс командной строки (CLI).
- **🔒 Полностью локальная работа** - Работает оффлайн, обеспечивая безопасность данных с помощью встроенных механизмов сетевой изоляции.
- **🔗 Работа в одном экземпляре** - Автоматически управляет экземплярами программы и поддерживает интеграцию с сопутствующим плагином Obsidian.

## 🚀 Быстрый старт

### Запуск программы

Для упакованной версии Windows: дважды щелкните `DocWen.exe`, чтобы запустить GUI. Для установки из исходников / через pip: запустите `docwen-gui`.

### Руководство по быстрому старту

1.  **Подготовьте файл Markdown**:

    ```markdown
    ---
    Заголовок: Тестовый документ
    ---
    
    ## Тестовый заголовок
    
    Это содержимое тела теста.
    ```

2.  **Конвертация перетаскиванием**:
    - Запустите программу.
    - Перетащите файл `.md` в окно.
    - Выберите шаблон.
    - Нажмите "Конвертировать в DOCX".

3.  **Получение результатов**:
    - Стандартизированный документ Word будет сгенерирован в том же каталоге.

**Совет**: Вы можете использовать примеры файлов в каталоге `samples/` для быстрого ознакомления с функциями программы.

## 📝 Соглашения о синтаксисе Markdown

### Сопоставление уровней заголовков

Чтобы облегчить запоминание коллегам без базовых знаний, заголовки Markdown в этом программном обеспечении соответствуют заголовкам Word **один к одному**:
- Заголовок документа (title) и подзаголовок (subtitle) помещаются в метаданные YAML.
- Markdown `# Заголовок 1` соответствует Word "Заголовок 1".
- Markdown `## Заголовок 2` соответствует Word "Заголовок 2".
- И так далее, поддерживается до 9 уровней заголовков.

**Совет**: Если вы предпочитаете использовать заголовок первого уровня Markdown (`#`) как заголовок документа, начиная со второго уровня (`##`) для подзаголовков текста, вы можете настроить стиль «Заголовок 1» в шаблоне Word так, чтобы он выглядел как заголовок документа (например, выравнивание по центру, жирный шрифт, более крупный размер шрифта), и выбрать схему нумерации, которая пропускает нумерацию заголовков первого уровня в настройках. Таким образом, ваши заголовки первого уровня будут выглядеть как заголовки документа.

### Разрывы строк и абзацы

**Основное правило**: Каждая непустая строка по умолчанию обрабатывается как отдельный абзац.

**Смешанные абзацы**: Когда подзаголовок должен быть смешан с основным текстом в одном абзаце, должны быть выполнены следующие условия:
1.  Подзаголовок заканчивается знаком препинания (поддерживаются многоязычные знаки препинания, включая точки, вопросительные знаки, восклицательные знаки и другие распространённые знаки завершения).
2.  Основной текст расположен на **следующей строке** после подзаголовка.
3.  Строка основного текста не может быть специальным элементом Markdown (таким как заголовки, блоки кода, таблицы, списки, цитаты, блоки формул, разделители и т.д.).

**Пример**:
```markdown
## I. Требования к работе.
Это совещание требует, чтобы все подразделения серьезно реализовали...
```
Вышеуказанные две строки будут объединены в один абзац, где "I. Требования к работе." сохраняет формат подзаголовка, а "Это совещание..." сохраняет формат основного текста.

**Примечание**:
- Между подзаголовком и основным текстом не должно быть пустой строки; в противном случае они будут распознаны как отдельные абзацы.
- Если подзаголовок не заканчивается знаком препинания и между ним и основным текстом нет пустой строки, основной текст будет объединен со строкой заголовка с измененным форматированием.

### Двусторонняя конвертация разделителей

Поддерживает двустороннюю конвертацию между разделителями Markdown и разрывами страниц/разрывами разделов/горизонтальными линиями Word:

-   **DOCX → MD**: Разрывы страниц, разрывы разделов и горизонтальные линии Word автоматически конвертируются в разделители Markdown.
-   **MD → DOCX**: Markdown `---`, `***`, `___` автоматически конвертируются в соответствующие элементы Word.
-   **Настраиваемость**: Конкретные отношения сопоставления могут быть настроены в интерфейсе настроек.

### Встраивание изображений и размер

Поддерживает встраивание изображений в стиле Obsidian/Wiki и стандартного Markdown, с опциональным размером (px):

```markdown
![[image.png]]
![[image.png|300]]
![[image.png\|300]]
![alt](image.png =300x200)
![alt](image.png =300x)
![alt|300](image.png)
```

- Без размера: исходный размер, ограничен доступной шириной (страница/ячейка)
- С размером: увеличение разрешено, но ширина всё равно ограничена
- Абзац только с изображением: использует стиль абзаца “Image” (по центру, одинарный интервал)

## 📖 Подробное руководство по использованию

### Word в Markdown

1.  Перетащите файл `.docx` в окно программы.
2.  Программа автоматически анализирует структуру документа.
3.  Генерирует файл `.md`, содержащий метаданные YAML.

**Поддерживаемые форматы**:
-   `.docx` - Стандартный документ Word.
-   `.doc` - Автоматически конвертируется в DOCX для обработки.
-   `.wps` - Документ WPS автоматически конвертируется.

**Опции экспорта**:

| Опция | Описание |
| :--- | :--- |
| **Извлечь изображения** | Если отмечено, изображения в документе извлекаются в выходную папку, а ссылки на изображения вставляются в файл MD. |
| **OCR изображений** | Если отмечено, выполняет OCR на изображениях и создает файл изображения .md (содержащий распознанный текст). |
| **Очистить номера подзаг.** | Если отмечено, удаляет номера перед подзаголовками (например, "一、", "（一）", "1." и т.д.) и конвертирует их в чистый текст заголовка. |
| **Добавить номера подзаг.** | Если отмечено, автоматически добавляет номера на основе уровней заголовков (схему нумерации можно настроить в настройках). |

### Markdown в Word

1.  Подготовьте файл `.md` с заголовком YAML.
2.  Перетащите его в окно программы и выберите соответствующий шаблон Word.
3.  Программа автоматически заполняет шаблон и генерирует документ.

**Опции конвертации**:

| Опция | Описание |
| :--- | :--- |
| **Очистить номера подзаг.** | Если отмечено, удаляет номера перед подзаголовками. |
| **Добавить номера подзаг.** | Если отмечено, автоматически добавляет номера на основе уровней заголовков. |

**Примечание**: Если в документе есть абзацы, где подзаголовки и основной текст смешаны, в файле MD должны соблюдаться строгие разрывы строк (см. "Разрывы строк и абзацы" выше).

### Автоматическая обработка стилей шаблона

Конвертер автоматически обнаруживает и обрабатывает стили шаблона во время конвертации Markdown → DOCX:

#### Классификация стилей

**Стиль абзаца**: Применяется ко всему абзацу.

| Стиль | Поведение обнаружения | Внедрение при отсутствии | Источник |
| :--- | :--- | :--- | :--- |
| Заголовок (1~9) | Обнаруживает стиль абзаца | Стили заголовков шаблона | Встроенный Word |
| Блок кода | Обнаруживает стиль абзаца | Шрифт Consolas + Серый фон | Определено ПО |
| Цитата (1~9) | Обнаруживает стиль абзаца | Серый фон + Левая граница | Определено ПО |
| Блок формулы | Обнаруживает стиль абзаца | Специфический стиль формулы | Определено ПО |
| Разделитель (1~3) | Обнаруживает стиль абзаца | Стиль абзаца нижней границы | Определено ПО |

**Стиль символа**: Применяется к выбранному тексту.

| Стиль | Поведение обнаружения | Внедрение при отсутствии | Источник |
| :--- | :--- | :--- | :--- |
| Встроенный код | Обнаруживает стиль символа | Шрифт Consolas + Серое затенение | Определено ПО |
| Встроенная формула | Обнаруживает стиль символа | Специфический стиль формулы | Определено ПО |

**Стиль таблицы**: Применяется ко всей таблице.

| Стиль | Поведение обнаружения | Внедрение при отсутствии | Источник |
| :--- | :--- | :--- | :--- |
| Трехлинейная таблица | Приоритет конфиг. польз. | Определение стиля трехлинейной таблицы | Определено ПО |
| Таблица-сетка | Приоритет конфиг. польз. | Определение стиля таблицы-сетки | Определено ПО |

**Определение нумерации**: Используется для форматов списков.

| Тип | Поведение обнаружения | Обработка при отсутствии |
| :--- | :--- | :--- |
| Нумерация списка | Сканирует существующие определения упорядоченных/неупорядоченных списков в шаблоне | Использует предустановку decimal/bullet |

#### Интернационализация имен стилей

-   **Встроенные стили Word** (heading 1~9):
    -   Имена стилей используют стандартные английские имена Word (например, `heading 1`).
    -   Word автоматически отображает локализованные имена в зависимости от языка системы (например, "Заголовок 1" в русской системе).
-   **Стили, определенные ПО** (Блок кода, Цитата, Формула, Разделитель, Таблица и т.д.):
    -   Внедряет соответствующие имена стилей языка в зависимости от настройки языка интерфейса ПО.
    -   Китайский интерфейс: Внедряет "代码块", "引用 1", "三线表" и т.д.
    -   Английский интерфейс: Внедряет "Code Block", "Quote 1", "Three Line Table" и т.д.

**Предложение**: После настройки стилей в шаблоне конвертер автоматически будет использовать ваши стили; если они отсутствуют в шаблоне, он будет использовать встроенные предустановленные стили.

### Обработка файлов таблиц

1.  **Excel/CSV в Markdown**: Перетащите файлы `.xlsx` или `.csv` для автоматической конвертации в таблицы Markdown.
2.  **Markdown в Excel**: Подготовьте файл MD и выберите шаблон Excel для конвертации.

**Поддерживаемые форматы**:
-   `.xlsx` - Стандартный документ Excel.
-   `.xls` - Автоматически конвертируется в XLSX для обработки.
-   `.et` - Таблица WPS автоматически конвертируется.
-   `.csv` - Текстовая таблица CSV.

### Функция корректуры текста

Программа предоставляет четыре настраиваемых правила корректуры:

1.  **Проверка парности пунктуации** - Обнаруживает, совпадают ли парные знаки препинания, такие как скобки и кавычки.
2.  **Коррекция символов** - Обнаруживает смешанное использование китайской и английской пунктуации.
3.  **Проверка опечаток** - Проверяет распространенные опечатки на основе пользовательского словаря.
4.  **Обнаружение чувствительных слов** - Обнаруживает чувствительные слова на основе пользовательского словаря.

**Пользовательские словари**: Визуально редактируйте словари опечаток и чувствительных слов в интерфейсе "Настройки".

**Использование**:
1.  Перетащите документ Word для проверки в программу.
2.  Отметьте необходимые правила корректуры.
3.  Нажмите кнопку "Корректура текста".
4.  Результаты корректуры отображаются в виде комментариев в документе.

## 🛠️ Система шаблонов

### Использование существующих шаблонов

Программа поставляется с различными шаблонами, включая многоязычные версии. Вы можете выбирать и использовать их по мере необходимости. Файлы шаблонов находятся в каталоге `templates/`.

### Пользовательские шаблоны

1.  Создайте файл шаблона с помощью Word или WPS.
2.  Обратитесь к существующим шаблонам и вставьте заполнители, такие как `{{Title}}` и т.д., где требуется заполнение.
3.  В шаблоне встроенные стили Заголовок 1 ~ Заголовок 5 должны быть изменены вручную.
4.  Сохраните шаблон в каталог `templates/`.
5.  Перезапустите программу, и новый шаблон будет загружен автоматически.

Вы также можете скопировать существующий шаблон, изменить его и переименовать.

### Использование заполнителей

#### Заполнители шаблона Word

**Заполнители полей YAML**: Используйте формат `{{ИмяПоля}}` в шаблоне, который будет заменен соответствующим значением в заголовке YAML файла Markdown во время конвертации.

| Заполнитель | Описание |
| :--- | :--- |
| `{{Заголовок}}` | Заголовок документа (Правила извлечения см. ниже) |
| `{{Тело}}` | Позиция вставки содержимого тела Markdown |
| Другие | Поддерживает любое пользовательское поле |

**Приоритет получения заголовка**:

| Приоритет | Источник | Описание |
| :--- | :--- | :--- |
| 1 | Поле YAML `Title` | Наивысший приоритет |
| 2 | Поле YAML `aliases` | Берет первый элемент списка или строковое значение |
| 3 | Имя файла | Имя файла без расширения `.md` |

**Многоязычная поддержка**: Заполнители заголовка и тела поддерживают несколько языков, например: заголовок может быть `{{Заголовок}}`, `{{title}}`, `{{标题}}` и т.д., тело может быть `{{Тело}}`, `{{body}}`, `{{正文}}` и т.д.

#### Заполнители шаблона Excel

Шаблоны Excel поддерживают три типа заполнителей:

**1. Заполнитель поля YAML** `{{ИмяПоля}}`

Используется для заполнения одного значения из заголовка YAML файла Markdown:

```markdown
---
ReportName: Статистика годовых продаж 2024
Unit: Отдел продаж
---
```

`{{ReportName}}`, `{{Unit}}` в шаблоне будут заменены соответствующими значениями. Поле заголовка также следует правилам приоритета.

**2. Заполнитель заполнения столбца** `{{↓ИмяПоля}}`

Извлекает данные из таблицы Markdown и заполняет **вниз** строку за строкой, начиная с позиции заполнителя:

```markdown
| ИмяПродукта | Количество |
|:--- |:--- |
| Продукт A | 100 |
| Продукт B | 200 |
```

`{{↓ИмяПродукта}}` в шаблоне Excel будет заменен на "Продукт A", а следующая строка будет заполнена "Продукт B".

**3. Заполнитель заполнения строки** `{{→ИмяПоля}}`

Извлекает данные из таблицы Markdown и заполняет **вправо** столбец за столбцом, начиная с позиции заполнителя:

```markdown
| Месяц |
|:--- |
| Янв |
| Фев |
| Мар |
```

`{{→Месяц}}` в шаблоне Excel будет заполнен "Янв", "Фев", "Мар" последовательно вправо.

**Обработка объединенных ячеек**: Программа автоматически пропускает непервые ячейки объединенных ячеек, чтобы обеспечить правильное заполнение данных.

**Слияние данных нескольких таблиц**: Если в Markdown есть несколько таблиц, использующих одно и то же имя заголовка, данные будут объединены по порядку и заполнены последовательно.

## 🖥️ Использование графического интерфейса

Большинство пользователей используют это программное обеспечение через графический интерфейс. Вот подробное руководство по эксплуатации.

### Обзор интерфейса

Программа использует **адаптивный трехколоночный макет**:

| Область | Описание | Время отображения |
| :--- | :--- | :--- |
| **Центральная колонка (Основная область)** | Область перетаскивания файлов, панель операций, строка состояния | Всегда отображается |
| **Правая колонка** | Выбор шаблона / Панель конвертации формата | Автоматически расширяется после выбора файла |
| **Левая колонка** | Список пакетных файлов (сгруппированных по типу) | Отображается при переключении в пакетный режим |

### Основной рабочий процесс

1.  **Запуск программы**: Дважды щелкните `DocWen.exe` (Windows пакет) или запустите `docwen-gui`.
2.  **Импорт файла**:
    -   Способ 1: Перетащите файлы прямо в окно.
    -   Способ 2: Нажмите кнопку "Добавить" в области перетаскивания, чтобы выбрать файлы.
3.  **Выбор шаблона** (если требуется конвертация): Правая панель шаблонов расширяется автоматически; выберите подходящий шаблон.
4.  **Настройка опций**: Отметьте необходимые опции конвертации/экспорта в панели операций.
5.  **Выполнение операции**: Нажмите соответствующую функциональную кнопку (например, "Экспорт MD", "Конвертировать в DOCX" и т.д.).
6.  **Просмотр результата**: Строка состояния показывает прогресс и результаты; нажмите значок 📍, чтобы найти выходной файл.

### Одиночный режим vs Пакетный режим

Программа поддерживает два режима обработки, переключаемых кнопкой-переключателем в области перетаскивания файлов:

**Одиночный режим** (По умолчанию):
-   Обрабатывает один файл за раз.
-   Простой интерфейс, подходит для ежедневного использования.

**Пакетный режим**:
-   Импортирует несколько файлов одновременно.
-   Левая колонка показывает категоризированный список файлов (сгруппированный по документу/таблице/изображению и т.д.).
-   Поддерживает пакетное добавление, удаление и сортировку.
-   Нажатие на файл в списке переключает текущую цель операции.

### Функции панели операций

Панель операций автоматически настраивает доступные опции в зависимости от типа файла:

| Тип файла | Доступные операции |
| :--- | :--- |
| Документ Word | Экспорт MD, Конвертировать PDF, Корректура текста, OCR |
| Markdown | Конвертировать DOCX, Конвертировать PDF |
| Таблица Excel | Экспорт MD, Конвертировать PDF, Сводка таблицы |
| Файл PDF | Экспорт MD, Слияние, Разделение, OCR |
| Файл изображения | Конвертация формата, Сжатие, OCR |

### Интерфейс настроек

Нажмите кнопку ⚙️ в правом нижнем углу окна, чтобы открыть настройки:

-   **Общие**: Тема интерфейса, язык, непрозрачность окна.
-   **Конвертация**: Значения по умолчанию для различных опций конвертации.
-   **Вывод**: Каталог вывода по умолчанию, правила именования файлов.
-   **Корректура**: Редактирование словарей опечаток и чувствительных слов.
-   **Стиль**: Конфигурации стиля блока кода, цитаты, таблицы.

### Горячие клавиши

-   **Перетаскивание внешнего файла**: Перетащите прямо в окно для импорта.
-   **Двойной щелчок по результату в строке состояния**: Быстро открыть каталог выходного файла.
-   **Щелчок правой кнопкой мыши по элементу шаблона**: Открыть расположение файла шаблона.

## 🔧 Использование командной строки

В дополнение к GUI, программа предоставляет интерфейс командной строки (CLI), подходящий для скриптов автоматизации и сценариев пакетной обработки.

### Режимы выполнения

-   **CLI режим**: Используйте подкоманды (например, `convert`, `validate`) для автоматизации и пакетной обработки.

### Распространенные примеры

```bash
# Пакетная версия (Windows)
DocWenCLI.exe convert document.docx --to md

# Экспорт Word в Markdown (Извлечь изображения + OCR)
DocWenCLI.exe convert report.docx --to md --extract-img --ocr

# Markdown в Word (Указать шаблон)
DocWenCLI.exe convert document.md --to docx --template "Название шаблона"

# Пакетная конвертация (Пропустить подтверждение, продолжить при ошибке)
DocWenCLI.exe convert *.docx --to md --batch --yes --continue-on-error

# Корректура документа
DocWenCLI.exe validate document.docx --check typo --check punct

# Слияние/Разделение PDF
DocWenCLI.exe merge-pdfs *.pdf
DocWenCLI.exe split-pdf report.pdf --pages "1-3,5,7-10"

# Из исходников / pip
docwen convert document.docx --to md
docwen convert report.docx --to md --extract-img --ocr
```

### Основные команды и опции

| Команда / Опция | Описание |
| :--- | :--- |
| `convert <files...> --to <fmt>` | Конвертировать в целевой формат (включая `md`) |
| `validate <files...> --check ...` | Проверить/корректировать документы (`--check typo/punct/symbol/sensitive/all/none`) |
| `merge-pdfs <files...>` | Объединить PDF/OFD/XPS файлы |
| `split-pdf <file> --pages ...` | Разделить PDF по диапазонам страниц |
| `merge-tables <files...> --mode row|col|cell` | Объединить таблицы |
| `merge-images-to-tiff <files...>` | Объединить изображения в TIFF |
| `md-numbering <files...>` | Обработать нумерацию заголовков Markdown |
| `templates list [--for docx|xlsx]` | Показать доступные шаблоны |
| `optimizations list [--scope ...]` | Показать доступные оптимизации |
| `formats list [--for-source document|spreadsheet|layout|image|markdown]` | Показать доступные целевые форматы |
| `inspect <file>` | Проверить категорию/формат и поддерживаемые действия |
| `--template <name>` | Имя шаблона (используется с `convert`) |
| `--extract-img` / `--no-extract-img` / `--ocr` | Опции для `convert --to md` |
| `--optimize-for <id>` | Явно включить оптимизацию (например `gongwen`, `invoice_cn`) |
| `--batch` / `--jobs` / `--continue-on-error` | Управление пакетной обработкой |
| `--json` | Вывести результат в формате JSON |
| `--quiet` / `-q` | Тихий режим, уменьшить вывод |
| `--lang` | Переключить язык (влияет на help/сообщения) |

### Основные характеристики

2.  **Клик снова (С файлом)** → Заменить новым файлом (Одиночный режим).
3.  **Клик снова (Без файла)** → Активировать окно конвертера.

## ❓ FAQ

### Что делать, если конвертация не удалась?

-   Проверьте, не занят ли файл другой программой.
-   Подтвердите, что формат файла правильный.
-   Проверьте логи ошибок в каталоге `logs/`.

### Шаблон не отображается?

-   Подтвердите, что файлы шаблонов находятся в каталоге `templates/`.
-   Проверьте, не поврежден ли файл шаблона.
-   Перезапустите программу, чтобы перезагрузить шаблоны.

### Функция корректуры не работает?

-   Подтвердите, что документ в формате .docx.
-   Проверьте, содержит ли документ редактируемый текст.
-   Подтвердите, что правила корректуры включены в настройках.

### Формат вывода не соответствует ожиданиям?

-   Программа генерирует документы на основе стилей шаблона. Чтобы настроить формат вывода, измените определения стилей непосредственно в файле шаблона.
-   Файлы шаблонов находятся в каталоге `templates/`.
-   После изменения стилей шаблона все документы, конвертированные с использованием этого шаблона, будут применять новые стили.

### Ячейки с формулами пусты после конвертации Excel в Markdown?

Это ожидаемое поведение. Программа читает **кэшированные значения** ячеек, а не сами формулы.

**Техническая причина**:
-   В файлах Excel ячейки с формулами хранят как формулу, так и последний вычисленный результат (кэшированное значение).
-   Программа использует режим `data_only=True`, который извлекает только кэшированные значения.
-   Если файл никогда не открывался в Excel (например, был сгенерирован программой) или был отредактирован, но не сохранен повторно, кэшированное значение будет пустым.

**Решение**:
1.  Откройте файл в Excel.
2.  Дождитесь завершения вычисления формул.
3.  Сохраните файл.
4.  Конвертируйте снова.

## 🔒 Функции безопасности

-   **Полностью локальная работа**: Вся обработка выполняется локально, нет зависимости от сети.
