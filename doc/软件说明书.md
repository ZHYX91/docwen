# 公文转换器软件说明书

## 软件概述

**软件名称**：公文转换器 (Gongwen Converter)  
**简称**：公文转换器  
**全称**：公文图表格式转换软件  
**版本号**：从 `src/gongwen_converter/__init__.py` 读取  
**开发语言**：Python 3.12+  
**运行环境**：Windows 10/11（主要支持），Linux/macOS（实验性支持）  
**作者**：ZhengYX  
**许可证**：GNU Affero General Public License v3.0 (AGPL-3.0)  
**项目主页**：https://github.com/ZHYX91/gongwen-converter

### 软件简介
公文转换器是一个专业的本地格式转换工具，支持公文与表格文件在 Word/Markdown/Excel 之间的多向转换。软件采用图形用户界面，操作简单直观，完全离线运行，确保数据安全。

### 主要用途
- 将Word公文文档转换为Markdown格式，便于内容管理和版本控制
- 将Markdown文档转换为格式规范的Word公文
- 处理Excel/CSV表格文件与Markdown之间的转换
- 自动转换DOC、WPS、XLS、ET等旧格式文件
- 智能校对公文中的错别字、标点、符号和敏感词问题
- 基于模板系统生成标准格式的公文

## 功能特点

### 核心功能

#### 1. Word转Markdown转换
- 智能解析公文格式结构
- 提取公文关键信息（标题、发文字号、主送机关等）
- 生成带YAML元数据的Markdown文件
- 支持Obsidian、Typora等Markdown编辑器
- 自动处理DOC、WPS格式（转为DOCX后处理）

#### 2. Markdown转Word转换
- 基于预定义模板生成规范公文
- 智能填充公文各要素
- 支持多种公文模板（通用公文、白头文、分局函等）
- 保持原有格式和样式

#### 3. 表格文件处理
- Excel/CSV转Markdown表格
- Markdown转Excel报表
- 自动处理XLS、ET旧格式（转为XLSX后处理）
- 支持复杂表格结构

#### 4. 文档校对系统
提供四种校对规则：

- **标点配对检查**：检测括号、引号等成对标点是否匹配
- **符号校对**：检测中英文标点混用问题
- **错别字检查**：基于自定义词典检查常见错别字
- **敏感词匹配**：检测文档中的敏感词使用

校对结果以Word批注形式标注问题位置和修改建议。

#### 5. 模板管理系统
- 内置多种常用公文模板
- 支持用户自定义模板
- 模板缓存机制提升性能
- 支持DOCX和XLSX两种模板格式

### 技术特点
- **完全离线运行**：所有操作在本地完成，无需网络连接
- **数据安全**：用户文件不上传至任何服务器，内置网络隔离机制（12个环境变量深度隔离）
- **高性能**：采用缓存和优化算法，转换速度快
- **易用性**：图形界面操作，拖拽即可转换
- **灵活配置**：9个TOML配置文件，支持深度定制
- **智能识别**：三轮评分算法识别公文元素，准确率高
- **OCR支持**：集成PaddleOCR，支持图片文字识别

## 使用说明

### 启动软件
1. 双击 `GongwenConverter.exe` 文件（可执行文件版）
2. 或运行 `python src/gui_run.py`（源码版）
3. 程序启动后显示主界面
4. 界面包含文件拖拽区域、模板选择器、功能按钮和状态栏

### Word转Markdown操作

#### 操作步骤
1. **拖拽文件**：将Word文档（.docx、.doc或.wps）拖入程序窗口中央的拖拽区域
2. **自动分析**：程序智能分析文档结构，识别公文元素
3. **等待完成**：状态栏显示转换进度
4. **查看结果**：在同目录下生成对应的Markdown文件（.md）

#### 生成文件说明
生成的MD文件包含两部分：
- **YAML头部**：包含公文元数据（标题、发文字号、主送机关等）
- **正文内容**：保持原有的标题层级和段落结构

示例：
```markdown
---
公文标题: 关于XXX的通知
发文字号: XX〔2024〕1号
主送机关: 各相关单位
成文日期: 2024年10月26日
---

## 一、通知事项

正文内容...
```

### Markdown转Word操作

#### 操作步骤
1. **准备MD文件**：确保MD文件包含正确的YAML头部
2. **拖拽文件**：将MD文件（.md或.txt）拖入程序窗口
3. **选择模板**：
   - 右侧模板选择器自动显示
   - 点击切换DOCX或XLSX模板类型
   - 选择具体模板（如"公文通用"）
4. **开始转换**：点击下方的"转换为DOCX"或"转换为XLSX"按钮
5. **查看结果**：在同目录下生成格式规范的Word文档

#### 注意事项
- YAML头部必须用 `---` 包围
- 如果公文中有小标题和正文混合，需在MD文件中保持严格换行
- 占位符（如`{{公文标题}}`）会被自动替换为实际内容

### 表格文件操作

#### Excel/CSV转Markdown
1. **拖拽表格文件**：将.xlsx、.xls、.et或.csv文件拖入程序
2. **自动转换**：程序自动将表格转为Markdown格式
3. **查看结果**：生成的MD文件包含标准Markdown表格语法

#### Markdown转Excel
1. **准备MD文件**：确保包含YAML头部和表格内容
2. **拖拽文件**：将MD文件拖入程序
3. **选择Excel模板**：在模板选择器中切换到XLSX类型
4. **转换**：点击"转换为XLSX"按钮
5. **查看结果**：生成的Excel文件保持原有格式

### 智能校对功能

#### 操作步骤
1. **打开文档**：将需要校对的Word文档（.docx）拖入程序
2. **选择规则**：在操作面板中勾选需要的校对规则：
   - ☑️ 标点配对检查
   - ☑️ 符号校对
   - ☑️ 错别字检查
   - ☑️ 敏感词匹配
3. **启动校对**：点击"智能校对"按钮
4. **等待处理**：状态栏显示校对进度
5. **查看结果**：
   - 生成带时间戳的新文档
   - 问题以Word批注形式标注
   - 批注包含错误类型、位置、建议修改

#### 批注说明
批注格式示例：
```
1. 错别字：因当 → 应当（上下文："...因当遵守..."）
2. 标点配对：缺少闭合引号（上下文："...关于"通知..."）
3. 敏感词：禁止（建议使用"不得"）
```

### 模板使用

#### 使用内置模板
程序自带多个常用公文模板：
- **公文通用.docx**：标准公文格式，适用于大多数公文
- **白头文.docx**：无红头公文，适用于内部文件
- **表格测试.xlsx**：Excel报表模板，适用于数据报表

#### 自定义模板

**创建新模板**：
1. 使用Word或WPS打开现有模板
2. 修改格式、样式、布局
3. 在需要填充的位置插入占位符：
   - `{{公文标题}}` - 公文标题
   - `{{发文字号}}` - 发文字号
   - `{{主送机关}}` - 主送机关
   - `{{成文日期}}` - 成文日期
   - `{{公文正文}}` - 正文内容
   - 其他自定义占位符
4. 保存为 `.docx` 或 `.xlsx` 格式
5. 将文件放入 `templates/` 目录
6. 重启程序，新模板自动加载

**占位符规则**：
- 使用双花括号包围：`{{字段名}}`
- 字段名与YAML头部中的键对应
- 特殊占位符`{{公文正文}}`用于插入整个正文

## 配置系统

### 配置文件说明

程序提供9个配置文件，位于 `configs/` 目录：

#### 1. gui_config.toml - 界面配置
```toml
[window]
default_width = 900
default_height = 600
auto_center = true

[theme]
default_theme = "cosmo"

[transparency]
enabled = true
default_value = 0.95
```

#### 2. symbol_settings.toml - 符号校对配置
```toml
[engine_settings]
enable_symbol_pairing = true
enable_symbol_correction = true

[symbol_pairing]
pairs = [
    ["（", "）"],
    ["《", "》"],
    [""", """]
]

[symbol_map]
"，" = ["。"]  # 中文逗号误用为句号
```

#### 3. typos_settings.toml - 错别字配置
```toml
[engine_settings]
enable_typos_rule = true

[typos]
"应当" = ["因当"]
"发文字号" = ["发文编号"]
```

#### 4. sensitive_words.toml - 敏感词配置
```toml
[engine_settings]
enable_sensitive_word = true

[sensitive_words]
"禁止" = ["明令禁止", "严禁"]
```

#### 5. conversion_config.toml - 转换配置
```toml
[output]
auto_open_folder = false

[cleanup]
delete_temp_files = true
```

#### 6. logger_config.toml - 日志配置
```toml
[logging]
level = "INFO"
max_file_size = 10485760  # 10MB
backup_count = 5
```

#### 7. link_config.toml - 链接嵌入配置
```toml
[format]
image_link_format = "wiki"
md_file_link_format = "wiki"

[embed_links]
enabled = true
image_mode = "embed"
md_file_mode = "embed"
```

#### 8. image_config.toml - 图片提取配置
```toml
[extraction_defaults]
docx_to_md_keep_images = true
docx_to_md_enable_ocr = false
xlsx_to_md_keep_images = true
layout_to_md_keep_images = true
```

#### 9. output_config.toml - 输出配置
```toml
[directory]
mode = "source"
create_date_subfolder = false

[behavior]
auto_open_folder = false

[intermediate_files]
save_to_output = true
```

## 常见问题解答

### 转换相关

**Q1：转换失败怎么办？**
- 检查文件是否被Word或WPS占用（关闭文件后重试）
- 确认文件格式正确（.docx、.md等）
- 查看 `logs/` 目录下的日志文件，了解详细错误信息
- 尝试将文件另存为新文件后再转换

**Q2：转换后格式不对？**
- 检查MD文件的YAML头部格式是否正确
- 确认选择了合适的模板
- 检查模板文件是否完整
- 查看日志了解具体问题

**Q3：表格转换后格式丢失？**
- 复杂表格可能需要手动调整
- 尝试使用不同的模板
- 检查原表格是否符合标准格式
- 某些特殊格式（如合并单元格）可能无法完美保留

### 模板相关

**Q4：模板不显示？**
- 确认模板文件在 `templates/` 目录中
- 检查模板文件扩展名正确（.docx或.xlsx）
- 检查模板文件是否损坏（尝试用Word打开）
- 重启程序重新加载模板列表

**Q5：如何添加自定义模板？**
- 参考现有模板创建新模板
- 使用 `{{占位符}}` 标记需要填充的位置
- 将模板文件保存到 `templates/` 目录
- 重启程序后在模板列表中选择

**Q6：占位符不工作？**
- 检查占位符格式：必须是 `{{字段名}}`
- 确保字段名与MD文件YAML头部的键完全一致
- 占位符区分大小写
- 检查没有多余的空格

### 程序运行

**Q7：程序无法启动？**
- **从源码运行**：
  - 确认 Python 版本为 3.12+：`python --version`
  - 重新安装依赖：`pip install .`
  - 检查是否有依赖冲突
- **可执行文件版**：
  - 确认系统为 Windows 10/11
  - 检查是否有足够的磁盘空间
  - 尝试以管理员权限运行
  - 查看 `logs/` 目录下的错误日志

**Q8：程序运行缓慢？**
- 关闭其他占用内存的程序
- 清理 `logs/` 目录中的旧日志文件
- 检查磁盘空间是否充足
- 尝试处理较小的文件进行测试

**Q9：日志文件太多？**
- 日志文件保存在 `logs/` 目录
- 可以手动删除旧日志文件
- 在 `logger_config.toml` 中调整日志保留策略
- 设置 `backup_count` 控制保留的日志文件数量

### 校对功能

**Q10：校对功能不工作？**
- 确认文档为 .docx 格式（不支持 .doc）
- 检查文档是否包含可编辑文本
- 在配置文件中确认相应的校对规则已启用
- 查看日志了解是否有错误信息

**Q11：校对结果不准确？**
- 这是基于规则的校对，不是AI校对
- 可以在配置文件中自定义错别字映射表
- 添加或修改 `typos_settings.toml` 中的规则
- 敏感词规则也可以自定义

**Q12：如何添加自定义错别字规则？**
1. 打开 `configs/typos_settings.toml`
2. 在 `[typos]` 节中添加规则：
   ```toml
   "正确写法" = ["错误写法1", "错误写法2"]
   ```
3. 保存文件并重启程序
