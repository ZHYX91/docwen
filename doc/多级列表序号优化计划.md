# 多级列表序号优化计划

## 背景

### 问题描述

当前问题：Word 文档中使用**多级列表**实现的小标题序号，在 DOCX→MD 转换时会丢失。

### 原因分析

```
Word 文档中的多级列表标题：
┌─────────────────────────────────────────┐
│ 段落样式: Heading 1                      │
│ para.text = "加强组织领导"               │ ← 不含序号
│ numPr: numId=1, ilvl=0                  │
│ 实际显示: "一、加强组织领导"             │ ← 序号来自 numbering.xml
└─────────────────────────────────────────┘
```

- `para.text` 不包含多级列表的序号
- 序号存储在 `numPr` 属性和 `numbering.xml` 中
- 当前代码只读取 `para.text`，导致序号丢失

### 相关概念

| 概念 | 说明 |
|------|------|
| 多级列表 | Word 中可定义多个级别的编号格式，支持关联样式 |
| numPr | 段落的列表属性，包含 `numId`（列表ID）和 `ilvl`（级别） |
| numbering.xml | 存储所有列表定义，包括编号格式和文本模板 |

---

## 短期计划：DOCX→MD 保留多级列表序号

### 目标

文档转 Markdown 时，正确提取并保留多级列表实现的小标题序号。

### 实现思路

```
1. 检测段落是否有 numPr 属性
   └─ 调用现有的 detect_list_item() 函数

2. 判断是否为公文标题格式
   └─ 根据 numFmt 判断（chineseCounting → 一级, decimal → 三级等）

3. 从 numbering.xml 获取列表格式定义
   ├─ abstractNumId → 找到 abstractNum 定义
   ├─ ilvl → 找到对应级别的 lvl 定义
   ├─ numFmt → 编号格式（chineseCounting, decimal 等）
   └─ lvlText → 文本模板（如 "%1、", "（%2）"）

4. 追踪各级别的编号计数器
   └─ 遍历文档时维护计数器状态

5. 重建完整的序号文本
   └─ 将计数器值转换为实际序号（如 1 → "一"）
```

### 涉及文件

| 文件 | 修改内容 |
|------|----------|
| `docx_utils.py` | 新增 `get_list_item_numbering()` 函数 |
| `converter_gongwen.py` | 在获取 `para_text` 后调用序号恢复逻辑 |
| `converter_simple.py` | 同步更新（如需要） |

### 技术要点

1. **解析 numbering.xml**
   - 获取 `abstractNum` 定义
   - 读取 `lvl` 中的 `numFmt` 和 `lvlText`

2. **支持多种 numFmt**
   | numFmt | 示例 | 对应标题级别 |
   |--------|------|-------------|
   | chineseCounting | 一、二、 | 1 |
   | ideographTraditional | （一）（二） | 2 |

   | 带括号 decimal | (1) (2) | 4 |
   | 带圈数字 | ① ② | 5 |

3. **维护列表计数器状态**
   - 按 `(numId, ilvl)` 分组计数

## 长期计划：MD→DOCX 序号使用多级列表实现

### 目标

Markdown 转 Word 时，小标题序号通过多级列表实现，而非文本拼接。

### 优势

| 当前方式（文本拼接） | 多级列表方式 |
|---------------------|-------------|
| 序号是文本的一部分 | 序号由 Word 管理 |
| 无法自动更新 | 添加/删除章节后自动重新编号 |
| 简单但不灵活 | 符合 Word 语义化结构 |

### 实现思路

```
1. 在模板中预定义公文专用多级列表
   ├─ 第1级: chineseCounting + "Heading 1"
   ├─ 第2级: ideographTraditional + "Heading 2"
   ├─ 第3级: decimal + "Heading 3"
   ├─ 第4级: decimal（带括号）+ "Heading 4"
   └─ 第5级: 带圈数字 + "Heading 5"

2. 修改 docx_processor.py
   ├─ 添加标题时不拼接序号文本
   ├─ 应用 Heading 样式（自动关联列表编号）
   └─ 或显式设置段落的 numPr 属性

3. 提供配置选项
   └─ numbering_method: "text" | "list"
```

### 涉及文件

| 文件 | 修改内容 |
|------|----------|
| 模板文件 | 添加公文专用多级列表定义 |
| `numbering_handler.py` | 扩展列表处理功能 |
| `docx_processor.py` | 修改标题写入逻辑 |
| `md_processor.py` | 调整序号处理流程 |

### 多级列表 XML 定义示例

```xml
<w:abstractNum w:abstractNumId="100">
  <!-- 第一级：一、二、三、 -->
  <w:lvl w:ilvl="0">
    <w:start w:val="1"/>
    <w:numFmt w:val="chineseCounting"/>
    <w:lvlText w:val="%1、"/>
    <w:pStyle w:val="Heading1"/>
  </w:lvl>
  
  <!-- 第二级：（一）（二）（三） -->
  <w:lvl w:ilvl="1">
    <w:start w:val="1"/>
    <w:numFmt w:val="ideographTraditional"/>
    <w:lvlText w:val="（%2）"/>
    <w:pStyle w:val="Heading2"/>
  </w:lvl>
  
  <!-- 第三级：1. 2. 3. -->
  <w:lvl w:ilvl="2">
    <w:start w:val="1"/>
    <w:numFmt w:val="decimal"/>
    <w:lvlText w:val="%3."/>
    <w:pStyle w:val="Heading3"/>
  </w:lvl>
  
  <!-- 第四级：(1) (2) (3) -->
  <w:lvl w:ilvl="3">
    <w:start w:val="1"/>
    <w:numFmt w:val="decimal"/>
    <w:lvlText w:val="(%4)"/>
    <w:pStyle w:val="Heading4"/>
  </w:lvl>
  
  <!-- 第五级：① ② ③ -->
  <w:lvl w:ilvl="4">
    <w:start w:val="1"/>
    <w:numFmt w:val="decimalEnclosedCircle"/>
    <w:lvlText w:val="%5"/>
    <w:pStyle w:val="Heading5"/>
  </w:lvl>
</w:abstractNum>
```
